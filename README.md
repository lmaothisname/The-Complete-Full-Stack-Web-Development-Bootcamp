## 🚀 How to Run a Project

Each project folder is self-contained. The general steps are:

### For static HTML/CSS/JS projects

Just open the `index.html` file in your browser:

```bash
cd "Introduction to HTML"
# Double-click index.html, or:
xdg-open index.html    # Linux
open index.html        # macOS
start index.html       # Windows
```

### For Node.js projects

```bash
cd NewsWeb             # or any Node project folder
npm install            # Install dependencies
npm run dev            # Start dev server (if available)
# OR
node index.js          # Run directly
```

Check the individual project's README (if available) for specific instructions.

## 💡 Key Takeaways

Some of the biggest lessons I've learned so far:

**On Front-End:** CSS is deceptively complex — understanding the box model, flexbox, and grid took more practice than I expected, but now layouts make sense.

**On JavaScript:** The language gets much easier once you understand that everything is an object and functions are first-class citizens. ES6 features like arrow functions, destructuring, and spread operators are game-changers.

**On Back-End:** Building a server with Express feels magical at first. Realizing that routes, middleware, and templates are just functions chained together made everything click.

**On Debugging:** The browser DevTools and `console.log` are your best friends. When code breaks, it's almost never the computer's fault — it's usually a typo or a misunderstood concept.
